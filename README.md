# Wildberries FBS Telegram Bot

Telegram-бот для автоматизации работы с поставками Wildberries по модели FBS (Fulfillment by Seller).

## Возможности

- 🔑 Сохранение и проверка API-ключа Wildberries
- 🔍 Автоматический мониторинг новых заказов (сборочных заданий)
- 📦 Создание поставок через API Wildberries
- 📋 Добавление товаров в поставку
- ✅ Подтверждение поставки и передача в доставку
- 📄 Генерация PDF со стикерами (QR-код поставки + штрих-коды заказов)
- 🖥 Удобные инлайн-кнопки для управления

## Требования

- Python 3.10+
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))
- API-ключ Wildberries (токен категории Marketplace)

## Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repo-url>
   cd wildberries_fbs_bot
   ```

2. **Создайте виртуальное окружение:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   venv\Scripts\activate     # Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте конфигурацию:**
   ```bash
   cp .env.example .env
   ```
   
   Отредактируйте `.env`:
   ```ini
   # Telegram Bot Token
   BOT_TOKEN=your_telegram_bot_token_here
   
   # ID офиса приёмки Wildberries (можно оставить 0 для ввода вручную)
   DESTINATION_OFFICE_ID=0
   
   # Интервал опроса новых заказов (в секундах)
   POLL_INTERVAL=30
   ```

5. **Запустите бота:**
   ```bash
   python bot.py
   ```

## Использование

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Начать работу с ботом |
| `/help` | Показать справку |
| `/set_key` | Сменить API-ключ |
| `/check` | Ручная проверка новых заказов |
| `/status` | Проверить статус подключения к API |

### Сценарий работы

1. **Регистрация:** Отправьте боту команду `/start` и введите API-ключ Wildberries
2. **Мониторинг:** Бот автоматически проверяет новые заказы каждые 30 секунд
3. **Новый заказ:** При появлении заказа бот отправляет уведомление с кнопкой «Создать поставку»
4. **Создание поставки:** Нажмите «Создать поставку», укажите ID офиса приёмки (если не задан в `.env`)
5. **Добавление товаров:** Нажимайте на товары, чтобы добавить их в поставку (или «Добавить все»)
6. **Подтверждение:** Нажмите «Подтвердить» для завершения поставки
7. **Стикеры:** Бот сгенерирует PDF с QR-кодом поставки и штрих-кодами заказов

## API Wildberries

Бот использует следующие методы API Wildberries (v3):

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/v3/orders/new` | Получение новых сборочных заданий |
| POST | `/api/v3/supplies` | Создание поставки |
| PATCH | `/api/marketplace/v3/supplies/{id}/orders` | Добавление заказов в поставку |
| GET | `/api/marketplace/v3/supplies/{id}/order-ids` | Список ID заказов в поставке |
| POST | `/api/v3/orders/stickers` | Получение стикеров заказов |
| PATCH | `/api/v3/supplies/{id}/deliver` | Подтверждение поставки |

## Обработка ошибок

- **401** — Неверный API-ключ
- **403** — Доступ запрещён
- **404** — Ресурс не найден
- **409** — Конфликт статусов (заказ уже в поставке)
- **429** — Превышение лимита запросов (rate limit)

## Примечания

- API-ключ хранится локально в `data/users.json` и не передаётся третьим лицам
- Для тестирования используйте API-ключ с Test Scope из песочницы Wildberries
- Стикеры заказов доступны только после добавления заказа в поставку (статус confirm)
- ID офиса приёмки можно найти в личном кабинете WB → Поставки → Офисы приёмки
